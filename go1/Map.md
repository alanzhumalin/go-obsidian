# Что такое Map?

Map - это структура данных, основанная на хеш-таблице, которая использует идею бакетов

![[Pasted image 20250313180258.png]]

- вставка
- удаление
- поиск по ключу

> [!Важно]
> Операции должны выполняться за константное время О(1)

---

### Структуры для работы с мапой:

``` Go
// go/src/runtime/map.gotype 
hmap struct {  
	// ...more code  
	count       int  
	B           uint 8  // также является capacity
	noverflow   uint 16  
	hash0       uint 32  
	buckets     unsafe.Pointer  
	oldbuckets  unsafe.Pointer
}
```

`count` — количество пар «ключ-значение».

`B` — количество бакетов, выраженное через log_2. Такое выражение через логарифм упрощает работу с битовыми операциями.

![[Pasted image 20250313180330.png]]

`noverflow` — приблизительное количество переполненных бакетов

`hash0` — величина для добавления большего разнообразия и случайности в хэш-функцию, позволяет достичь устойчивости к коллизиям.

`buckets` — это указатель на массив бакетов. В Go это ячейка памяти, где хранятся бакеты.

`oldbuckets` — это также указатель на массив бакетов, из которых происходит эвакуация данных, когда бакетов становится больше. Является ячейкой памяти.

### Создание мапы

Мы можем объявить переменную с типом `map[string]string` двумя способами:

1. `var m map[string]string`
2. `m := make(map[string]string)`

Рассмотрим создание map с использованием функции `make`. При создании map через make используется функция `makemap`.

``` Go
  func makemap(t *maptype, hint int, h *hmap) *hmap {    
  //...more code
  
  // Block 1  
  // initialize Hmap
	  if h == nil {      
		  h = new(hmap)    
	  }    
	  h.hash0 = fastrand()

	// Block 2
	  B := uint8(0)    
	  for overLoadFactor(hint, B) {      
		  B++    
	  }    
	  h.B = B3   

	// Block 3
	  if h.B != 0 {
		var nextOverflow *bmap      
		h.buckets, nextOverflow = makeBucketArray(t, h.B, nil) //...more code
	}    
	return h  
}
```

**Блок 1.** Проверяем наличие `h *hmap`. Если его нет, то инициализируем. Затем получаем hash0.

**Блок 2.** Рассчитываем количество бакетов _B_ в формате _log_ на основе `hint`. `hint` — второй аргумент в функции `make(map[string]string, hint)`.

**Блок 3.** Если количество бакетов не равно 1 (_log 1 = 0 = B_), то выделяется память для наших бакетов.

# Ключи в мапе
### **Условия, при которых структура может быть ключом:**

1. **Типы всех полей должны быть сравнимыми**:
    
    - Типы, которые можно использовать для сравнения: числа, строки, булевые значения, указатели, каналы, массивы (если их элементы сравнимы).
    - Нельзя использовать: слайсы, карты, функции. Эти типы несравнимы.
2. **Две структуры сравнимы, если:**
    
    - Они имеют одинаковую структуру (поля и их типы).
    - Все их поля равны.

---

### **Пример: структура как ключ в `map`:**

```go
package main

import "fmt"

// Определяем структуру
type Point struct {
    X int
    Y int
}

func main() {
    // Создаём map с ключами-структурами
    points := make(map[Point]string)

    // Добавляем элементы
    points[Point{X: 1, Y: 2}] = "Точка A"
    points[Point{X: 3, Y: 4}] = "Точка B"

    // Доступ по ключу
    fmt.Println(points[Point{X: 1, Y: 2}]) // Вывод: Точка A
    fmt.Println(points[Point{X: 3, Y: 4}]) // Вывод: Точка B
}
```

---

### **Когда структура НЕ может быть ключом?**

Если структура содержит **несравнимые типы**, она не может быть использована в качестве ключа.

Пример:

```go
type InvalidKey struct {
    Data []int // Слайсы несравнимы
}

func main() {
    // Это вызовет ошибку компиляции:
    // invalid map key type InvalidKey
    // invalidKeyMap := make(map[InvalidKey]string)
}
```

---

### **Обходные пути для использования несравнимых типов в ключах:**

1. **Сериализация структуры в строку**: Преобразуй структуру в строку (например, с помощью `fmt.Sprintf`) и используй эту строку как ключ:
    
    ```go
    type Data struct {
        Slice []int
    }
    
    func main() {
        data := make(map[string]string)
    
        key := fmt.Sprintf("%v", Data{Slice: []int{1, 2, 3}})
        data[key] = "Value for Data"
    
        fmt.Println(data[key]) // Вывод: Value for Data
    }
    ```
    
2. **Использование хеширования**: Для сложных структур можно использовать хеш-функции для создания уникальных строк или чисел, которые будут ключами.
    

---

### **Итог:**

- Структура может быть ключом в `map`, если все её поля имеют сравнимые типы.
- Несравнимые типы (например, слайсы, карты) делают структуру непригодной для использования в качестве ключа.
- В обход можно использовать сериализацию или хеширование, если нужно использовать сложные структуры в качестве ключей.
