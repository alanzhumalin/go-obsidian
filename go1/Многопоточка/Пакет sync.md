Пакет **`sync`** в Go предоставляет примитивы синхронизации для управления доступом к общим ресурсам в многопоточных приложениях. Он используется для обеспечения потокобезопасности и предотвращения гонок данных (data races).

---

### **Основные компоненты пакета `sync`**

#### 1. **`sync.Mutex`**

- Примитив для обеспечения взаимного исключения (mutual exclusion).
- Используется для защиты общих данных от одновременного доступа из нескольких горутин.

**Пример использования:**

```go
package main

import (
	"fmt"
	"sync"
)

func main() {
	var mu sync.Mutex
	count := 0

	// Инкремент из нескольких горутин
	for i := 0; i < 10; i++ {
		go func() {
			mu.Lock()              // Блокировка
			count++
			mu.Unlock()            // Разблокировка
		}()
	}

	// Ждём, чтобы горутины завершились
	fmt.Scanln()
	fmt.Println("Итоговое значение count:", count)
}
```

- **`Lock()`**: блокирует ресурс.
- **`Unlock()`**: освобождает ресурс.
- Используй `defer` для автоматической разблокировки:
    
    ```go
    mu.Lock()
    defer mu.Unlock()
    ```
    

---

#### 2. **`sync.RWMutex`**

- Расширение `Mutex`, позволяющее разделять блокировки на:
    - **Чтение** (`RLock`) — несколько горутин могут одновременно читать данные.
    - **Запись** (`Lock`) — только одна горутина может записывать.

**Пример использования:**

```go
package main

import (
	"fmt"
	"sync"
)

func main() {
	var rwMu sync.RWMutex
	data := 0

	// Горутина для чтения
	go func() {
		rwMu.RLock()
		fmt.Println("Чтение данных:", data)
		rwMu.RUnlock()
	}()

	// Горутина для записи
	go func() {
		rwMu.Lock()
		data = 42
		fmt.Println("Данные обновлены")
		rwMu.Unlock()
	}()

	fmt.Scanln()
}
```

#### 3. **`sync.WaitGroup`**

- Используется для ожидания завершения группы горутин.
- Позволяет избежать ручного подсчёта горутин с использованием `sync.Mutex`.

**Пример использования:**

```go
package main

import (
	"fmt"
	"sync"
)

func main() {
	var wg sync.WaitGroup

	// Запускаем 5 горутин
	for i := 1; i <= 5; i++ {
		wg.Add(1) // Увеличиваем счётчик

		go func(id int) {
			defer wg.Done() // Уменьшаем счётчик после завершения
			fmt.Printf("Горутина %d завершилась\n", id)
		}(i)
	}

	wg.Wait() // Ожидаем завершения всех горутин
	fmt.Println("Все горутины завершены")
}
```

---

#### 4. **`sync.Once`**

- Позволяет выполнить определённый код **только один раз**, даже если он вызывается из нескольких горутин.

**Пример использования:**

```go
package main

import (
	"fmt"
	"sync"
)

func main() {
	var once sync.Once

	init := func() {
		fmt.Println("Этот код выполняется только один раз")
	}

	for i := 0; i < 3; i++ {
		go once.Do(init)
	}

	fmt.Scanln()
}
```

#### 5. **`sync.Map`**

- Потокобезопасная структура данных для работы с `map`.
- Подходит для сценариев, где нужно избежать гонок данных при доступе к карте.

**Методы:**

- **`Store(key, value)`** — сохранить значение.
- **`Load(key)`** — получить значение.
- **`Delete(key)`** — удалить ключ.
- **`Range(func(key, value interface{}) bool)`** — перебрать все элементы.

**Пример использования:**

```go
package main

import (
	"fmt"
	"sync"
)

func main() {
	var sm sync.Map

	// Сохраняем данные
	sm.Store("name", "John")
	sm.Store("age", 30)

	// Получаем данные
	name, _ := sm.Load("name")
	fmt.Println("Name:", name)

	// Перебираем все элементы
	sm.Range(func(key, value interface{}) bool {
		fmt.Printf("%v: %v\n", key, value)
		return true
	})
}
```


---

### **Когда использовать пакет `sync`?**

1. Для защиты общих данных от одновременного доступа из нескольких горутин.
2. Для управления синхронизацией между горутинами.
3. Для потокобезопасных структур данных (например, `sync.Map`).
4. Когда требуется гарантированное выполнение кода (например, `sync.Once`).
