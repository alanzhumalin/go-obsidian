# Стек и Куча в Go

***Стек (Stack)*** — представь, что ты кладёшь тарелки одна на другую. Берёшь сверху, кладёшь сверху. Последняя положенная тарелка — первая, которую возьмёшь. Это принцип ***LIFO (Last In, First Out).***  

***Куча (Heap)*** — это как если бы тарелки лежали хаотично в шкафу. Ты можешь взять любую, но придется её поискать.  

Как это работает в Go?  
- **Стек** → Быстрая работа, но ограниченный размер памяти.  
- **Куча** → Дольше живут данные, но работает медленнее.  

Пример стека в Go:  
```Go
func printMessage() {
    msg := "Привет, мир!" // В стеке
    fmt.Println(msg)
}

func main() {
    printMessage()
}
```

✅ msg создаётся и уничтожается автоматически, как только функция завершится.  

Пример кучи в Go:

```Go
func createMessage() *string {
    msg := "Привет, мир!"
    return &msg // Go переносит в кучу
}

func main() {
    message := createMessage()
    fmt.Println(*message)
}
```

✅ Go понимает, что msg нужна за пределами функции → она отправляется в кучу.  

Вывод:   
- Если данные нужны только в функции — они в стеке.  
- Если должны жить дольше или передаются по указателю — в куче.

# Когда Go кладёт данные в стек, а когда в кучу?  

### Когда данные попадают в стек?  

Если переменная:  
- Локальная и объявлена внутри функции.  
- Не передаётся по указателю.  
- Небольшая, Go понимает, что её можно безопасно удалить после выхода из функции.   

```Go
func add(a int) int {
    b := a + 10 // Всё в стеке
    return b
}

func main() {
    x := 42
    y := add(x) // Тоже стек
    println(y)
}
```

✅ Здесь всё в стеке, потому что переменные не выходят за границы функций и/или они небольшого размера.  

### Когда данные попадают в кучу?  

Если переменная:  
- Создаётся через указатель (&) и возвращается из функции.  
- Передаётся в другую горутину.  
- Является большой структурой, чтобы не переполнить стек.  
- Go сам решает вынести её в кучу (escape analysis).  

```Go
func createPointer() *int {
    x := 100
    return &x // x уходит в кучу
}

func main() {
    p := createPointer() // p теперь указывает на x в куче
    println(p)
}
```

✅ Go понимает, что x должна жить дольше createPointer(), поэтому переносит её в кучу.

# Где хранятся данные в компьютере?  

![[Pasted image 20250313193317.png]]

Память делится на 4 уровня:  

Жёсткий диск (HDD, SSD)  
- Медленный, но самый большой по объёму.  
- Здесь хранятся файлы, базы данных, операционная система.  
- Пример:  
  - Файл базы данных PostgreSQL (https://t.me/c/2228290676/41/157) (перейти (https://t.me/+cOH3whU7UJNiZWNi))  
  - Образы Docker (https://t.me/c/2228290676/41/158)-контейнеров (перейти (https://t.me/+cOH3whU7UJNiZWNi))

Оперативная память (RAM)  
- Быстрее диска, но объём ограничен.  
- Хранятся исполняемые программы, кешированные данные.  
- Кэшированные базы данных (Redis)
- В RAM находятся:  
  - Куча  (Heap) → для динамически выделяемой памяти (make, new).  
  - Стек  (Stack) → для локальных переменных функций.  

Кэш процессора (L3 → L2 → L1)  
- Очень быстрый, но маленький по объёму.  
- Используется для кеширования часто используемых данных.  
- Примеры:
  - L3 – общий для всех ядер процессора (например, последние инструкции).  
  - L2 – хранит часто используемые блоки данных.  
  - L1 – самый быстрый, используется для хранения инструкций прямо перед их выполнением.  

Регистры процессора  
- Самая быстрая, но самая маленькая память.  
- Используются процессором для выполнения инструкций.
