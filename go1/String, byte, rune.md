# Конспект по Go: типы, string, byte, rune, Unicode/UTF-8

## 1) Какие типы данных есть в Go

### 1.1 Предобъявленные (встроенные, predeclared)

- bool
- string
- int, int8, int16, int32, int64
- uint, uint8, uint16, uint32, uint64, uintptr
- float32, float64
- complex64, complex128
- error (интерфейс)

### 1.2 Составные (composite)

- array ([N]T)
- slice ([]T)
- map (map[K]V)
- struct
- pointer (*T)
- func
- interface
- chan

### 1.3 “Примитивные/непримитивные”

- Это неофициальная терминология в Go (учебная).
- Обычно:
- “примитивные” = bool, числа, string
- “непримитивные” = array/slice/map/struct/pointer/func/interface/chan

---

## 2) Что такое alias (алиас)

Алиас = другое имя **того же** типа.

`type MyByte = uint8`

MyByte и uint8 полностью одинаковы для компилятора.

Не путать с новым типом:

`type UserID int64 // новый тип, не алиас`

Для нового типа обычно нужны явные преобразования.

---

## 3) byte и rune

- byte = алиас uint8
- rune = алиас int32

Смысл:

- byte используют для работы с байтами (файлы, сеть, JSON body, бинарные данные).
- rune используют для работы с Unicode code point (условно “символами”).

---

## 4) Unicode, ASCII, UTF-8

- Unicode = система символов + кодовых точек (U+...).
    
- Кодовая точка = числовой ID символа.
    
- Примеры:
    
- A → U+0041
    
- Ж → U+0416
    
- Қ → U+049A
    
- 🙂 → U+1F642
    
- ASCII = старый набор из 128 символов.
    
- UTF-8 = способ кодировать Unicode в байты (1–4 байта на code point).
    
- ASCII полностью совместим с UTF-8 (для ASCII символов байты одинаковые).
    

---

## 5) Что под капотом у string в Go

- string всегда хранит **неизменяемую последовательность байтов**.
- Тип string не меняется на rune, даже если там кириллица/emoji.
- В строковых литералах Go эти байты обычно UTF-8.
- Но string может хранить и невалидный UTF-8 (это тоже допустимо как набор байтов).

---

## 6) len, индексация, []byte, []rune

`s := "Привет" fmt.Println(len(s)) // 12 (байты) fmt.Println(len([]rune(s))) // 6 (code points) fmt.Println(s[0]) // первый байт (byte), не первая буква`

- len(string) считает байты.
- s[i] возвращает байт (byte).
- []byte(s) дает UTF-8 байты строки.
- []rune(s) декодирует строку в Unicode code points.

---

## 7) Разбор твоего примера (alan, Алан, Қазақ)

Почему так:

- "alan": ASCII, 1 буква = 1 байт, len=4.
- "Алан": 4 буквы, каждая по 2 байта UTF-8, len=8.
- "Қазақ": 5 букв, обычно по 2 байта UTF-8, len=10.

Почему string(r[0]) дал Ð, Ò:

- r[0] это только первый байт UTF-8 последовательности, не целая буква.
- Поэтому получаешь “кракозябру”.

Почему se[0] = 'X' дало X�азақ:

- Ты заменил только 1 байт внутри многобайтной буквы.
- Получилась невалидная UTF-8 последовательность, появляется �.

Почему с []rune получилось Xазақ:

- Замена идет по code point, а не по отдельному байту.

---

## 8) Важный нюанс про “символ”

rune = code point, но это не всегда “один видимый символ пользователя”.  
Примеры:

- эмодзи-флаг может состоять из нескольких code points
- буква + комбинируемый диакритический знак = несколько rune

---

## 9) Практические правила (когда что брать)

- string:
    
- хранение/передача текста
    
- JSON/API поля, логирование, UI-текст
    
- []byte:
    
- I/O, сеть, файлы, шифрование/хеширование
    
- когда нужно менять байты или работать с сырыми данными
    
- []rune:
    
- когда нужно менять/считать текст по Unicode code points

## 11) Итог одной фразой

string в Go всегда байты; byte — работа с байтами; rune — работа с Unicode code points; len(string) считает байты, не “человеческие символы”.
