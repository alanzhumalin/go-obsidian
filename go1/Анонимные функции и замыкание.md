# Зачем нужны анонимные функции?

(в проектах они почти никогда не используются. все предпочитают именные функции, но на собесе может попасться)

1. Для одноразового использования:
   - Когда не нужно создавать отдельную именованную функцию.  
   - Экономит место, если функция маленькая и простая.
2. Для передачи как аргумент:  
   - Полезно для сортировки, фильтрации и работы с коллекциями.
3. Для создания замыканий:  
   - Анонимные функции могут ***"запоминать"*** данные из внешней области видимости.

```Go
package main

import "fmt"

func main() {
    sum := func(a, b int) int {
        return a + b
    }
    
    result := sum(3, 5)
    fmt.Println("Результат:", result) // Выведет: Результат: 8
}
```

Сортировка с `sort.Slice`
```Go
package main

import (
  "fmt"
  "sort"
)

func main() {
  numbers := []int{5, 2, 9, 1, 4}

  // Сортировка с анонимной функцией
  sort.Slice(numbers, func(i, j int) bool {
    return numbers[i] < numbers[j] // Условие сортировки
  })

  fmt.Println(numbers) // [1 2 4 5 9]
}
```

- Здесь функция сравнения используется только для сортировки, поэтому давать ей отдельное имя нет смысла.

# Замыкание

***Замыкание*** — это функция, которая ***"запоминает"*** переменные из внешней области видимости, даже после того, как эта область завершила выполнение.
Эти переменные не исчезают после выхода из функции, а ***"живут"*** вместе с замыканием.

```Go
func counter() func() int {
  count := 0
  return func() int {
    count++
    return count
  }
}

func main() {
  next := counter()

  fmt.Println(next()) // 1
  fmt.Println(next()) // 2
  fmt.Println(next()) // 3
}
```

`counter()` создаёт локальную переменную `count`.
Возвращаемая анонимная функция ***"помнит"*** значение `count`, даже после выхода из `counter()`.
Каждый вызов `next()` увеличивает `count`, потому что замыкание сохраняет его состояние.

## Зачем нужно замыкание?

- Переменные остаются ***"скрытыми"*** внутри функции, доступ к ним возможен только через замыкание.
- Можно создавать функции с уникальной логикой на основе внешних данных

```Go
func multiplier(factor int) func(int) int {
    return func(x int) int {
        return x * factor
    }
}

func main() {
    double := multiplier(2)
    triple := multiplier(3)

    fmt.Println(double(5)) // 10
    fmt.Println(triple(5)) // 15
}
```

Здесь `double` и `triple` — это разные замыкания, каждое из которых помнит своё значение `factor`.
