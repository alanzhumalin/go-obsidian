# Замыкание (closure) в Go

Замыкание в Go это функция, которая использует переменные из внешней области видимости и “помнит” их между вызовами.

## Коротко
- Замыкание = `функция` + `захваченные внешние переменные`.
- В Go захватывается сама переменная, а не её “копия по умолчанию”.
- Часто это анонимная функция, возвращённая из другой функции.

## Пример 1: состояние внутри функции (счётчик)
```go
package main

import "fmt"

func Counter(start int) func() int {
	n := start
	return func() int {
		n++
		return n
	}
}

func main() {
	next := Counter(10)
	fmt.Println(next()) // 11
	fmt.Println(next()) // 12
	fmt.Println(next()) // 13
}
```

## Где используется
- Callback-функции (обработчики, хуки, ретраи).
- Middleware (например, HTTP-обёртки с конфигом).
- Фабрики функций (создаём функцию с “преднастроенными” параметрами).
- Goroutine/`defer`, когда нужно использовать контекст внешних переменных.

## Пример 2: фабрика фильтра
```go
package main

import "fmt"

func GreaterThan(limit int) func(int) bool {
	return func(x int) bool {
		return x > limit
	}
}

func main() {
	gt10 := GreaterThan(10)
	fmt.Println(gt10(7))  // false
	fmt.Println(gt10(15)) // true
}
```

## Важный нюанс (циклы)
В старом Go была частая ошибка с захватом переменной цикла в goroutine. Надёжный стиль:
```go
for i := 0; i < 3; i++ {
	i := i // локальная копия для текущей итерации
	go func() {
		fmt.Println(i)
	}()
}
```

Итог: замыкание полезно, когда функция должна хранить контекст/состояние без отдельной структуры.
